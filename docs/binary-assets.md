# Binary assets and release archives

The PR review flow used for this project does not support binary file diffs. Do
not commit generated archives, image assets, icon files, or built executables to
source PRs.

Use these rules instead:

- Build output such as `*.zip`, `*.exe`, `*.sha256`, helper binaries, and release
  manifests should be generated by CI or local release scripts and attached to a
  GitHub Release.
- Image and icon assets such as `*.png` and `*.ico` must be added only through a
  binary-capable release/artifact process, or replaced with source-friendly text
  formats such as SVG when they need code review.
- PyInstaller builds must not require source-committed binary logo files. The GUI
  already treats optional logo files as best-effort runtime assets when present.
- If a future release needs signed/logo-bearing Windows artifacts, keep those
  binaries out of the PR diff and document the external asset source in the
  release notes.
